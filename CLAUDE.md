## Project Overview

DE424 Mini-Project: Wumpus Tracker using Hidden Markov Model (HMM) inference via Belief Propagation.

The program estimates the trajectory of a Wumpus moving on a grid, given noisy binary sensor observations at each timestep. Three datasets are supported with varying grid sizes and parameters.

## Branch Policy

- **`belief`** — active development branch. All new changes go here.
- **`prod`** — clean production snapshot. Branched from `belief`. No output files, no build artifacts.
- **`main`** — base branch. Do not push directly to `main`.
- **Never switch branches unless explicitly instructed.**

## CLI Interface

```
wumpus <dataset_dir> <dataset_id> <out_prefix> [--pw v] [--pc v] [--em n]
```

- `dataset_id` 1, 2, or 3 — selects grid size and default parameters
- `out_prefix` — output files written as `<prefix>_marginal.txt`; the Makefile passes `results/out_dN` so all outputs land in `results/`
- `--pw`, `--pc` — override emission probabilities
- `--em n` — override EM iteration count

## Dataset Parameters

| Dataset | Grid | Timesteps | pw | pc | EM |
|---------|------|-----------|-----|-----|----|
| 1 | 5×5 | 10 | 0.95 | 0.05 | No |
| 2 | 20×20 | 20 | 0.90 | 0.10 | No |
| 3 | 10×20 | 20 | learned | learned | Yes (6 iters) |

## Key Design Decisions

- **No compile-time dataset flags.** Dataset selection is entirely runtime via CLI `argv[2]`. There is no `ACTIVE_DATASET` macro or `#ifdef` switching.
- **Single output file per run:** `_marginal.txt` — argmax of per-timestep marginals P(X_t | Z_1:T).
- **All outputs go to `results/`**, which is gitignored and created automatically by Make before any run target.
- **EM is only enabled for dataset 3** via `cfg.learn_pwpc = true` and `cfg.max_iters = 6`.
- **emdw** is the probabilistic toolkit providing `DiscreteTable`, `ClusterGraph`, and `LBP_CG`. Do not modify files under `emdw_de424/`.
- **CMakeLists** (`emdw_de424/devel/emdw/src/bin/CMakeLists.txt`) points to `tests/basic/` and `tests/advanced/` for all test executables.

## Files NOT in Repo (gitignored)

- `results/` — entire output directory: `out_dN_marginal.txt`, `out_dN_anim.gif`. Regenerated by `make run`.

## Build & Run

```bash
make build        # Configure and build with CMake
make run          # Run all 3 datasets, write outputs to results/
make demo         # Run all datasets with verbose output + trajectory preview
make run-d1       # Run dataset 1 only
make accuracy     # Score predictions vs ground truth
make clean        # Remove results/ directory
```

## Test Targets

```bash

make test-advanced  # Run advanced tests (test_e2e_sim, test_dataset2_scale, test_em_extremes)
```
