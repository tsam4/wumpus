## Project Overview

DE424 Mini-Project: Wumpus Tracker using Hidden Markov Model (HMM) inference via Belief Propagation.

The program estimates the trajectory of a Wumpus moving on a grid, given noisy binary sensor observations at each timestep. Three datasets are supported with varying grid sizes and parameters.

## Branch Policy

## CLI Interface

```
wumpus <dataset_dir> <dataset_id> <out_prefix> [--pw v] [--pc v] [--em n]
```

- `dataset_id` 1, 2, or 3 ‚Äî selects grid size and default parameters
- `out_prefix` ‚Äî output files written as `<prefix>_marginal.txt`
- `--pw`, `--pc` ‚Äî override emission probabilities
- `--em n` ‚Äî override EM iteration count

## Dataset Parameters

| Dataset | Grid | Timesteps | pw | pc | EM |
|---------|------|-----------|-----|-----|----|
| 1 | 5√ó5 | 10 | 0.95 | 0.05 | No |
| 2 | 20√ó20 | 20 | 0.90 | 0.10 | No |
| 3 | 10√ó20 | 20 | learned | learned | Yes (6 iters) |

## Key Design Decisions

- **No compile-time dataset flags.** Dataset selection is entirely runtime via CLI `argv[2]`. There is no `ACTIVE_DATASET` macro or `#ifdef` switching.
- **Single output file per run:** `_marginal.txt` ‚Äî argmax of per-timestep marginals P(X_t | Z_1:T).
- **EM is only enabled for dataset 3** via `cfg.learn_pwpc = true` and `cfg.max_iters = 6`.
- **emdw** is the probabilistic toolkit providing `DiscreteTable`, `ClusterGraph`, and `LBP_CG`. Do not modify files under `emdw_de424/`.

## Build & Run

```bash

make run          # Run all 3 datasets, generate outputs
make demo         # Run all datasets with verbose output
make run-d1       # Run dataset 1 only
make accuracy     # Score dataset 1 vs ground truth
make clean        # Remove output txt files
```
