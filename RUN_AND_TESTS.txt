Run + Test Guide (post-execution)

This file documents exactly how the program was run, how to interpret outputs, how to run the tests, and the observed results from this run on 6 May 2026. All model assumptions and dataset parameters are taken from mini_project.pdf.

1) How to run the program
Executable: wumpus (built inside emdw)
CLI:
  wumpus <dataset_dir> <dataset_id> <out_prefix> [--pw v] [--pc v] [--em n] [--occ prior]

Required args:
- dataset_dir: path to a dataset folder containing data_file*.txt grids
- dataset_id: 1..5 (selects known parameters or enables EM/occupancy)
- out_prefix: output file prefix

2) File roles (what does what)
- src/wumpus.cc: main program. Builds factors, runs BP, updates parameters, writes outputs.
- src/wumpus_model.cc: shared helpers for IO, transitions, emissions, Viterbi MAP.
- include/wumpus.hpp: shared structs (Grid, DatasetConfig).
- include/wumpus_model.hpp: shared helper function declarations.
- tests_basic/: minimal unit-level tests for core constraints.
- tests_advanced/: stress and edge-case tests.

3) Output files and interpretation
Program outputs two files per run:
- <out_prefix>_marginal.txt: per-time-step argmax of marginals p(X_t | Z_1:T)
- <out_prefix>_map.txt: MAP path from Viterbi (max-product equivalent)

Each line is "x y" (x = column, y = row), zero-indexed, top-left origin, matching the ground-truth format in mini_project.pdf (see dataset1 ground truth).

4) How to run tests
All tests were built and run inside emdw as separate executables. Commands used:

Build:
  cd ~/devel/emdw/build
  cmake ../
  make -j7 wumpus test_transition test_emission test_parser test_emdw_bp \
           test_dataset2_scale test_em_extremes test_occ_prior_stress test_e2e_sim

Run:
  src/bin/test_transition
  src/bin/test_emission
  src/bin/test_parser
  src/bin/test_emdw_bp
  src/bin/test_dataset2_scale /Users/taros/Documents/DE424MP/Datasets-20260506/dataset2
  src/bin/test_em_extremes
  src/bin/test_occ_prior_stress
  src/bin/test_e2e_sim

Interpreting test outputs:
- Most tests produce no output if all assertions pass.
- test_dataset2_scale prints a single line with runtime and memory stats:
  "dataset2 bp_ms=<ms> map_ms=<ms> maxrss=<value>"
  On macOS, ru_maxrss is in bytes; on Linux it is typically in kilobytes.
